Mysql的一些知识点

发表于 2020-06-14 | 分类于 技术 | 阅读次数:

事务的四大特性

  • 原子性: 事务要么全部成功,要么全部不成功
  • 隔离性: 事务之间的操作是彼此独立的.
  • 一致性: 如果数据库在执行事务前是一致的,那么在执行操作后也是一致的.
  • 持久性: 事务一旦提交,它的结果就是永久的

事务的隔离级别

  1. read uncommitted

    最低级别,如果一个改变了数据库的值,即使事务未提交,其他事务也会感受到.可能导致脏读.

    比如,A给B的余额转1000块钱,过程中发现只需要转100,立刻回滚事务,但是B在这个过程中会看到自己的余额多了1000.这就是脏读

  2. read commited

    一个事务必须等另一个事务提交了才能读到数据,可以避免脏读.

    比如,A要付款1000,他查了下余额刚好够1000,但是同时他的妻子有事把他的钱转出去了,然后A付款的时候,钱就不够了.A付款的时候发现余额不足了,一次事务里两次读到的数据不一样.

  3. repeatable read

    可以避免不可重复读问题,就是在A在进行查询余额操作后,不允许其它事务再修改A的余额了,只有A付完款,事务结束后,其他的事务才能修改A的余额.

    但是,B事务又可能会在A事务的数据里插入新的字段,A两次的到的数据字段数量都不一样了,这就是幻读.

    mysql默认是这一级别.

  4. Serializable 序列化

    这是隔离的最高级别,事务串行化执行,可以避免脏读、不可重复读和幻读.但是会损失一部分效率.

B+树

要了解B+树得先了解B树,B树是一种多路平衡搜索树,每个节点可以存多个值.
B+树的数据只存放在叶子节点上,不存在于内部节点上,并且叶子节点还有一条链表连起来,所以,可以很方便的遍历叶子结点.

为什么用B+树来存索引

mysql中的数据是持久化在磁盘中的,访问磁盘数据的操作分为定位和读取两个部分.磁盘中有两个机械运动的部分,盘片旋转和磁臂移动.盘片旋转就是我们提到的磁盘转速,在磁盘转到指定位置后,磁臂会向前移动,读取指定位置数据的读取.定位是一个很花费时间的过程,用B树可以优化它.

  1. B+树的高度比二叉树低,定位到叶子节点更快.更快的查找到数据.
  2. B+树和B相比,非叶子节点不保存数据,占用空间小.磁盘同一盘块可容纳的节点多,定位时需要移动的位置就小很多.
  3. B+树的数据都放在叶子节点,搜索比较稳定.
  4. B+树的叶子节点之间由链表连接,比较方便扫库,遍历链表就行了.

mysql索引类型

  • index 普通索引 数据可以重复
  • unique 唯一索引 数据不可以重复
  • peimary key 主键索引 数据不可重复不可为空,一个表只能有一个主键
  • 组合索引 在多个字段上创建索引,左前缀匹配
  • fulltext 全文索引 主要用来查询是否包含某个词
