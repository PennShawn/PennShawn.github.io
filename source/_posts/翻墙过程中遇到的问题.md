在destroy服务器后,又购买了同一个ip的服务器,使用相同的
ssh root@45.77.42.213
命令后,提示挺吓人的
```
 ✘ playcrab@shenpeng  ~  ssh root@45.77.42.213
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@    WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!     @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY!
Someone could be eavesdropping on you right now (man-in-the-middle attack)!
It is also possible that a host key has just been changed.
The fingerprint for the ECDSA key sent by the remote host is
SHA256:zeSvrx3IMQvopctfr0VxMzxKIm0//Vh6/bCm0xNv5+4.
Please contact your system administrator.
Add correct host key in /Users/playcrab/.ssh/known_hosts to get rid of this message.
Offending ECDSA key in /Users/playcrab/.ssh/known_hosts:12
ECDSA host key for 45.77.42.213 has changed and you have requested strict checking.
Host key verification failed.
```
根据https://www.cnblogs.com/huiy/p/6230913.html的解释,远程服务器重装系统了,需要清除当前机器里关于远程服务器的缓存和公钥信息.
使用命令 
```
 ✘ playcrab@shenpeng  ~  ssh -keygen -R 45.77.42.213
Bad escape character 'ygen'.
 ✘ playcrab@shenpeng  ~  ssh -key gen -R 45.77.42.213
Bad remote forwarding specification '45.77.42.213'
 ✘ playcrab@shenpeng  ~  ssh -key gen -R "45.77.42.213"
Bad remote forwarding specification '45.77.42.213'
 ✘ playcrab@shenpeng  ~  ssh-keygen -R "45.77.42.213"
# Host 45.77.42.213 found: line 12
/Users/playcrab/.ssh/known_hosts updated.
Original contents retained as /Users/playcrab/.ssh/known_hosts.old
```
之后,再重新登录即可
````
 playcrab@shenpeng  ~  ssh root@45.77.42.213
The authenticity of host '45.77.42.213 (45.77.42.213)' can't be established.
ECDSA key fingerprint is SHA256:zeSvrx3IMQvopctfr0VxMzxKIm0//Vh6/bCm0xNv5+4.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added '45.77.42.213' (ECDSA) to the list of known hosts.
root@45.77.42.213's password:
````
